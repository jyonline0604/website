# GitHub 提交規則

## 📋 基本原則

### 1. 所有數據必須真實
- ✅ 天氣數據：香港天文台官方 API
- ✅ 交通數據：運輸署官方 API  
- ✅ 公共交通：data.gov.hk 官方 API
- ✅ 新聞數據：Google News RSS 實時數據
- ❌ 禁止使用測試數據或模擬數據

### 2. 自動記憶記錄
- 每次重要修改後自動記錄到 `memory/YYYY-MM-DD.md`
- GitHub 推送後記錄提交詳情
- 每天結束前更新長期記憶（MEMORY.md）

### 3. 自動 GitHub 推送
- 修改完成後立即推送到 GitHub
- 無需用戶提醒
- 用戶可以在 GitHub 直接查看效果

## 📝 提交信息格式

### 提交類型（必須使用以下前綴）：
- `feat:` - 新功能
- `fix:` - 修復問題
- `docs:` - 文檔更新
- `style:` - 代碼格式（不影響功能）
- `refactor:` - 代碼重構
- `test:` - 測試相關
- `chore:` - 構建過程或輔助工具

### 提交信息格式：
```
type: 簡短描述

- 詳細說明修改內容
- 列出修改的文件
- 說明修改原因
- 如果有相關問題，說明解決方案
```

### 示例：
```
feat: 添加真實財經新聞系統

- 創建 real_finance_news.py 模塊
- 獲取真實加密貨幣和美股新聞
- 數據來源：Google News RSS 實時數據
- 確保財經新聞100%真實
- 文件：scripts/real_finance_news.py
```

## 📁 文件組織規則

### 目錄結構：
```
workspace/
├── scripts/          # 所有腳本文件
│   ├── novel/       # 小說相關腳本
│   ├── briefing/    # 簡報相關腳本
│   ├── news/        # 新聞相關腳本
│   └── utils/       # 工具腳本
├── my-novel/        # 小說網站文件
├── memory/          # 每日記憶文件
└── docs/            # 文檔文件
```

### 文件命名：
- 使用小寫字母和連字符：`generate-briefing.py`
- 描述性名稱：`real_finance_news.py`
- 避免通用名稱：`test.py`、`script.py`

## 🧪 代碼質量要求

### 1. 代碼規範：
- Python：使用 PEP 8 規範
- Shell：使用 ShellCheck 檢查
- 適當的註釋和文檔

### 2. 錯誤處理：
- 所有 API 調用必須有錯誤處理
- 使用日誌記錄重要事件
- 實現備用方案（fallback）

### 3. 安全性：
- 不提交敏感信息（API 密鑰、密碼）
- 使用環境變量或配置文件
- 定期安全檢查

## 🔄 工作流程

### 修改流程：
1. **計劃**：明確修改目標和要求
2. **開發**：編寫代碼，確保質量
3. **測試**：測試功能是否正常
4. **記錄**：自動記錄到記憶檔
5. **提交**：按照格式提交到 Git
6. **推送**：自動推送到 GitHub
7. **驗證**：在 GitHub 驗證效果

### 緊急修復：
- 問題發現後立即修復
- 優先修復影響用戶的功能
- 記錄問題原因和解決方案

### 推送後檢查：
- [ ] GitHub 提交成功
- [ ] 文件正確上傳
- [ ] 記憶檔更新提交記錄

## 🚨 禁止事項

### 絕對禁止：
- ❌ 提交虛假或測試數據
- ❌ 提交敏感信息
- ❌ 破壞性修改不經測試
- ❌ 忽略錯誤處理
- ❌ 忘記記憶記錄

### 需要批准：
- ⚠️ 重大架構變更
- ⚠️ 數據源變更
- ⚠️ 自動化規則修改
- ⚠️ 安全相關修改
